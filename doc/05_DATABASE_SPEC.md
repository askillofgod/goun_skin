---

# 1. 문서 목적

본 문서는 고운통증케어 홈페이지의 데이터 저장 구조를 정의한다.

본 프로젝트는 데이터베이스(DB)를 사용하지 않는 정적(Static) 웹사이트로 구축한다.

---

# 2. 데이터베이스 사용 여부

## 사용 여부

❌ 사용하지 않음

---

## 사용 이유

고운통증케어 홈페이지는

- 회원가입 기능 없음
- 로그인 기능 없음
- 관리자 페이지 없음
- 자체 예약 시스템 없음
- 온라인 결제 없음

구조이므로 데이터베이스가 필요하지 않다.

---

# 20. 최종 결론

고운통증케어 홈페이지는

예약 및 문의를 외부 플랫폼(네이버 예약, 카카오톡)에 위임하는 구조로 설계한다.

따라서 데이터베이스를 사용하지 않으며,

모든 콘텐츠는 정적 데이터 파일로 관리한다.

이는 구축 비용과 유지보수 비용을 최소화하면서도 충분한 예약 전환 효과를 제공한다.
